# クロスハッチ陰影 for YMM4

[![License](https://img.shields.io/badge/license-MIT-blue.svg)](#)
[![.NET](https://img.shields.io/badge/.NET-10.0-purple.svg)](#)
[![Release](https://img.shields.io/github/v/release/routersys/YMM4-CrossHatching.svg)](https://github.com/routersys/YMM4-CrossHatching/releases)

---

YukkuriMovieMaker4（YMM4）上で動作する、**入力映像の輝度と局所的な明暗勾配からクロスハッチ線を手続き的に生成し、ペン画・版画・漫画・設計図風の陰影へ変換する映像エフェクトプラグイン**です。
参照画像・深度画像・機械学習モデルを使わず、単一の入力フレームだけで陰影を成立させます。暗い部分ほど線の層が段階的に増え、流れの値に応じて線方向が明暗の勾配へ沿います。
数値パラメータはアニメーションに対応しています。

## 概要

本プラグインは YMM4 の「映像エフェクト」として動作し、エフェクトの種類一覧では「クロスハッチ陰影」として表示されます。

Direct2D カスタムピクセルシェーダーがフレームごとに入力映像の輝度を求め、暗さに応じて最大 4 方向の線層を重ねます。中心画素とその周囲 8 画素から Scharr 係数で輝度・アルファの勾配を計算し、等輝度線の接線方向へ線を寄せることで、曲面に沿った陰影を表現します。線は周期パターンとして手続き的に生成され、外部テクスチャを必要としません。粒状感はシードに基づく決定論的ハッシュで生成するため、静止映像ではフレームごとにちらつきません。

このエフェクトは AviUtl 向けの EXO 出力（`.exo`）には対応していません。

## インストール方法

1. [Releases](https://github.com/routersys/YMM4-CrossHatching/releases/latest) ページから最新のプラグインファイル（`.ymme`）をダウンロードしてください。
2. YMM4 が起動していないことを確認し、ダウンロードしたファイルを実行（ダブルクリック）してインストールします。
3. YMM4 を起動し、タイムライン上のアイテムに映像エフェクトを追加します。
4. 映像エフェクトの種類として **「クロスハッチ陰影」** を選択してください。

## 主な機能

### 1. スタイルプリセット

ペン画・版画・漫画・設計図の 4 種類のスタイルを用意しています。スタイルを切り替えると、線の方向・暗部しきい値・粒状感・輪郭の既定構成が内部で切り替わります。最初に触るべき項目はスタイル・密度・太さ・陰影の 4 つです。

### 2. 輝度に応じた線層

入力の暗さ（`1 − 輝度`）と陰影パラメータから陰影量を求め、最大 4 層の線を段階的に重ねます。白に近い部分では線がほぼ出ず、暗い部分ほど線層が増えます。線の層の最大数は線の層パラメータで制限でき、自動では 4 層をしきい値で制御します。

### 3. 明暗勾配に沿う線方向

中心画素と周囲 8 画素から輝度の勾配を求め、その接線方向（等輝度線方向）へ線を寄せます。流れが 0% のときはスタイル固定の方向だけを使い、流れを上げるほど勾配へ沿います。勾配が十分に小さい平坦部では固定方向へ戻し、方向の不安定化を防ぎます。線層ごとに 0°・90°・45°・-45° のオフセットを加えるため、曲面上でも交差したハッチが得られます。

### 4. 輪郭と粒状感

輝度とアルファの勾配の大きさから輪郭層を生成し、ハッチとは別に重ねます。輪郭が 0% のときは輪郭を出力しません。粒状感はシードに基づくハッシュで生成し、線のかすれ（インクのかすれ）と紙のノイズに反映します。シードが同じであれば、同じ入力の同じ座標で粒状感は常に同じ結果になります。

### 5. 色モード

出力色の作り方を 2 種類から選べます。インクと紙では紙色とインク色だけで描き、元の色を残すでは元映像の色を残してインク線を重ねます。いずれのモードでも入力のアルファを維持し、透明部分は透明のまま出力します。

## パラメータ一覧

### 基本グループ

| パラメータ名 | 型 | デフォルト | スライダー表示範囲 | アニメーション | 説明 |
|---|---|---|---|---|---|
| スタイル | 列挙 | ペン画 | ペン画 / 版画 / 漫画 / 設計図 | ✗ | 線の方向と階調と質感のプリセットを選びます。 |
| 適用量 | 数値 | 100% | 0 〜 100% | ✔ | 元映像とハッチ結果の合成量です。0% では入力を変更しません。 |
| 密度 | 数値 | 12px | 3 〜 80px | ✔ | 線の間隔です。小さいほど線が密になります。 |
| 太さ | 数値 | 1.2px | 0.2 〜 12px | ✔ | 各ハッチ線の太さです。 |
| 陰影 | 数値 | 100% | 0 〜 200% | ✔ | 暗部で線が増える強さです。 |
| 流れ | 数値 | 40% | 0 〜 100% | ✔ | 明暗の勾配へ線の向きを寄せる強さです。 |
| 輪郭 | 数値 | 30% | 0 〜 100% | ✔ | ハッチに重ねる輪郭線の強さです。 |

### 色グループ

| パラメータ名 | 型 | デフォルト | 説明 |
|---|---|---|---|
| インク色 | 色 | `#FF111111` | ハッチ線と輪郭線の色です。 |
| 紙色 | 色 | `#FFF6F1E7` | 紙または背景の色です。 |
| 色 | 列挙 | インクと紙 | 出力色の作り方です。インクと紙 / 元の色を残す から選びます。 |

### 詳細グループ

| パラメータ名 | 型 | デフォルト | スライダー表示範囲 | アニメーション | 説明 |
|---|---|---|---|---|---|
| 線の層 | 列挙 | 自動 | 自動 / 1 / 2 / 3 / 4 | ✗ | 暗部に重ねる線方向の最大数です。 |
| 階調の柔らかさ | 数値 | 12% | 0 〜 50% | ✔ | 線層が現れる境界の柔らかさです。 |
| 粒状感 | 数値 | 12% | 0 〜 100% | ✔ | 紙とインクの手続き的な粒状感です。 |
| シード | 整数 | 0 | 0 〜 9999 | ✗ | 粒状感と線位相を固定する値です。 |

## スタイルプリセットの内訳

スタイルごとに、線の方向（4 層）・暗部しきい値（4 層）・粒状感と輪郭の倍率が切り替わります。

| スタイル | 線の方向 | 暗部しきい値 | 粒状感倍率 | 輪郭倍率 |
|---|---|---|---|---|
| ペン画 | -45° / 45° / 0° / 90° | 0.25 / 0.45 / 0.65 / 0.82 | 0.60 | 1.00 |
| 版画 | 20° / -20° / 70° / -70° | 0.18 / 0.36 / 0.56 / 0.76 | 1.00 | 0.60 |
| 漫画 | -45° / 45° / 0° / 90° | 0.30 / 0.50 / 0.70 / 0.86 | 0.50 | 1.40 |
| 設計図 | 0° / 90° / 45° / -45° | 0.22 / 0.44 / 0.66 / 0.84 | 0.35 | 1.40 |

版画のみ、線が交差した箇所の濃度をわずかに強めます。

## 制限事項

- AviUtl 向けの EXO 出力（`.exo`）には対応していません。
- 透明部分（アルファが 0）には効果が適用されません。

---

## 注意事項

- **AviUtl 非対応**: 本プラグインは AviUtl 向けの EXO 出力（`.exo`）に対応していません。
- **透明部分の扱い**: ソース画像の透明部分（アルファが 0）はそのまま出力され、線や輪郭が透明外周へ漏れません。
- **シードと再現性**: シードが同じであれば、粒状感と線の位相は常に同じ結果になります。
- **本プラグインを使用する前に、YMM4 プロジェクトファイルのバックアップを作成することを推奨します。**
